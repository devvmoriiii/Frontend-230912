캐스케이딩 (Cascading)

: 스타일 시트에서는 우선 순위가 위에서 아래로, 계단식으로 적용 CSS 에서는 웹 요소에 둘 이상의 스타일을 적용할 때 우선순위에 따라 적용할 스타일을 결정
'스타일 우선순위' 와 '스타일 상속' 으로 나뉨.

스타일 우선순위

: '중요도', '적용 범위', '소스코드의 작성 순서' 에 따라 우선순위를 결정

중요도 : 웹 브라우저에 내용을 표시할 때 단순히 css 소스의 스타일만 적용되지 않음.

1 > 2 > 3 순서로 적용도가 높다.

  1. 사용자가 지정한 스타일 (ex: 윈도우의 고대비 모드를 설정 : 검은색)
  2. 제작자(브라우저를 사용하는 사람) 스타일 (ex: 제작자가 구글 페이지를 초록색으로 지정 : 초록색)
  3. 브라우저 기본 스타일 (ex: 구글 페이지의 기본 색상 : 흰색)

적용 범위 : 중요도가 같은 스타일일 경우 적용 범위에 따라 우선순위를 결정
스타일 적용 범위가 좁을수록 (정확히 필요한 요소에만 적용할수록) 우선 순위가 높아짐.

1 > 2 > 3 > 4 > 5 순서로 적용도가 높다.

  1. !important : 어떤 스타일보다 우선 적용
  2. 인라인 스타일 : 태그 안에 style 속성을 사용하여 해당 태그만 스타일을 적용
  3. id 스타일
  4. class 스타일
  5. 타입 (= 태그, 요소) 스타일

소스 코드의 작성 순서 : 중요도와 적용 범위가 같은 스타일일 경우, 소스 코드의 순서로 우선순위를 결정
나중에 작성한 스타일이 먼저 작성한 스타일을 덮어쓴다.

스타일 상속

: 태그의 포함 관계에 따라 부모 요소의 스타일을 자식 요소로, 위에서 아래로 전달

상속되는 속성 vs 상속되지 않는 속성

상속되는 속성

  • 텍스트 관련 속성 (Ex. color, font-, 텍스트 정렬 등)
  • 리스트 관련 속성 (Ex. list-style 등)

상속되지 않는 속성

  • 박스 모델 속성 (Ex. 높이, 너비, padding, margin, border 등)
  • 배경 스타일 속성 (Ex. background 등) 단, 배경 속성의 기본값은 투명이므로 상속된 것처럼 보이지만, 상속된 형태가 아님.
  • 위치, 디스플레이 속성 (Ex. display, position, top, left 등)

강제 상속

: inherit 키워드를 사용하여 강제 상속 가능하다.
