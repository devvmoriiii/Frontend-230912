Form 태그

form 태그 : 사용자로부터 정보를 입력받기 위한 인터페이스로, 사용자가 웹 사이트로 정보를 보낼 수 있는 요소

- 텍스트 필드, 체크박스, 라디오 버튼 등의 다양한 유형의 입력 요소를 위한 컨테이너
- 폼 자체는 표시되지 않음
Ex. 아이디, 비밀번호 입력, 로그인 버튼, 회원가입 버튼 등

- 입력 받은 자료를 어떤 형식으로 서버에 전송할 것인지, 어떤 프로그램을 이용해 처리할 것인지를 지정

form 태그의 속성

  • method 속성 : 사용자가 입력한 내용을 서버 프로그램에 어떻게 넘겨줄 것인지 지정

    get : URL 에 폼 데이터를 포함시켜 전송하며, 데이터를 전송하는 길이를 제한(256 ~ 4,096 byte)

    주소 표시줄에 사용자가 입력한 내용이 그대로 표시되므로 보안성이 낮음

    post : 데이터를 요쳥 본문에 포함시켜 전송하며, 민감한 데이터 전송이나 데이터 변경 작업에 사용

    입력한 내용의 길이 제한이 없고 사용자가 입력한 내용도 드러나지 않음

  • action 속성 : 폼 데이터를 처리할 서버의 URL 을 지정
  • target 속성 : action 속성에서 지정한 파일을 현재 창이 아닌 다른 위치에서 열도록 지정
  • autocomplete 속성 : 자동 완성 기능을 나타내는 속성 (기본값 : on) / off 로 변경 시,

    • 폼 요소 그룹화 및 레이블 적용

    fieldset & legend : 폼 요소를 그룹으로 묶는 태그

    fieldset 태그 : 하나의 폼 안에서 여러 구역을 나누어 포함시키는 태그

    legend 태그 : fieldset 으로 묶은 그룹에 제목을 붙이는 태그
